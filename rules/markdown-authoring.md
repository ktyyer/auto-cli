# Markdown 文件编写规范

本仓库是纯 Markdown 指令仓库，所有产物均为 .md 文件。

## 最小 diff 原则

- 修改时只改需要改的行，不顺手重构无关内容
- 不新增无根据的兜底说明或兼容描述
- 不添加与当前需求无关的示例、注释或解释

## 文档与行为一致性

- 文档必须以当前真实行为为准，不能承诺尚未实现的功能
- 发现文档与实际行为冲突时，优先修文档到一致
- 移除功能后同步删除对应文档，不留过时引用

## 格式要求

- 运行 `npm run format:check` 确保格式一致（Prettier 校验）
- 修改子命令后检查内部引用是否一致（文件路径、Agent 名、Skill 名）

## 子命令自包含

- 子命令 md 文件是自包含的指令，不依赖外部 JS 运行时
- 子命令中引用的 Agent/Skill 必须在 agents/ 或 skills/ 中实际存在
- 子命令 frontmatter 遵循 Claude Code slash command 格式
