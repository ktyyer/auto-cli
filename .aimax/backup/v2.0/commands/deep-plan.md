# Deep Plan 命令

`/aimax:deep-plan` 用于复杂任务的两阶段规划：先深度探索代码库，再用干净上下文执行。

---

## 核心理念

传统做法：探索代码库 → 积累大量上下文 → 直接编码 → **上下文污染导致质量下降**

Deep Planning：探索代码库 → 生成精炼计划 → **丢弃探索上下文** → 用干净上下文执行

---

## 两阶段流程

### 阶段一：深度探索（Plan Mode）

在此阶段，**禁止修改任何文件**。只做以下事情：

1. **代码库扫描**
   - 分析项目结构、模块依赖、关键路径
   - 识别代码模式、命名规范、架构风格
   - 发现相关文件和调用链

2. **需求澄清**
   - 提出有针对性的问题，消除歧义
   - 确认约束条件和边界

3. **生成 `implementation_plan.md`**
   - 精确的文件修改清单
   - 每个修改的具体策略
   - 依赖顺序和风险评估
   - 测试覆盖策略

### 阶段二：干净执行（Act Mode）

1. **加载精炼计划** — 只读 `implementation_plan.md`，不加载阶段一的探索上下文
2. **逐步实现** — 按计划执行，每步写测试
3. **门禁验证** — 构建、测试、lint 通过

---

## 输出模板

### `implementation_plan.md` 格式

```markdown
# 实现计划：[任务标题]

## 目标
- ...

## 代码库分析摘要
- 关键模块：...
- 依赖关系：...
- 代码风格：...

## 修改清单

### 1. [文件路径]
- 修改类型：新增 / 修改 / 删除
- 具体变更：...
- 原因：...

### 2. [文件路径]
- ...

## 执行顺序
1. → 2. → 3. ...

## 风险与缓解
- 风险：...
- 缓解：...

## 测试策略
- 单元测试：...
- 集成测试：...
```

---

## 何时使用

- 任务涉及 5+ 个文件修改
- 需要理解复杂的模块依赖关系
- 需要重构或架构级变更
- 对代码库不熟悉，需要先探索

---

## 与其他命令协作

- `/aimax:deep-plan` → 生成精炼计划
- `/aimax:loop` → 用状态机编排执行
- `/aimax:tdd` → 在执行阶段保证测试覆盖
- `/aimax:code-review` → 完成后审查

---

## 使用示例

```bash
/aimax:deep-plan 重构用户认证模块，从 session 迁移到 JWT
```

```bash
/aimax:deep-plan 将订单系统拆分为微服务
```
