# PRD 标准结构 + 三种模式详情

> 完整 PRD 文档结构模板和三种生成模式的详细执行流程。

## PRD 标准结构

```markdown
# [产品名称] PRD

## 1. 产品概述

- 产品背景
- 核心价值
- 目标用户

## 2. 用户场景

- 用户故事
- 使用流程

## 3. 功能需求

### 3.1 核心功能

### 3.2 功能流程图

## 4. 交互细节

### 4.1 正常流程

### 4.2 异常情况

### 4.3 状态定义

## 5. 字段规范

- 字段名、类型、必填、校验规则

## 6. 文案规范

- 按钮文案、提示文案、错误文案

## 7. 视觉稿

- 线框图链接
- 流程图链接

## 8. 技术约束

- 依赖系统
- 性能要求
```

## 模式一：从零生成

### 第一阶段：概念版（对齐方向）

三个提问视角：

1. **用户视角** — 目标用户是谁？怎么解决？痛点在哪？
2. **商业视角** — 一次性还是长期？商业模式？
3. **技术视角** — 产品形态？开发成本？

输出：1 页概念版 PRD + 待确认问题列表

### 第二阶段：落地板（补全细节）

五个必填要素：

1. **交互细节** — 加载/空/错误状态 + 异常路径
2. **状态机** — 字段/按钮所有状态 + 转换关系
3. **字段规范** — 必填/长度/校验规则
4. **文案规范** — 统一按钮和提示文案
5. **流程图** — 核心用户流程 + 关键页面线框图

## 模式二：优化已有文档

1. 分析现有文档 → 找出遗漏细节和结构问题
2. 补充五个必填要素
3. 输出一致性格式

## 模式三：增加新需求

1. 定位新功能与现有功能的关系
2. 影响分析
3. 整合输出

## 使用示例

**用户**: "我想做一个自媒体账号定位工具"

**AI 提问**（概念版）:

1. 目标用户是谁？新手博主还是成熟创作者？
2. 用户现在怎么解决账号定位问题？
3. 数据从哪取？

→ 产出概念版 PRD + 待确认问题

**AI 深化**（落地板）:
基于回答补全交互细节、状态机、字段规范...

→ 最终产出可开发 PRD
