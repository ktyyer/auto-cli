# PRD Writer 参考模板

> 本文档包含两阶段详细提问模板和完整 PRD 结构示例。

---

## 阶段一：概念版提问模板

### 用户视角提问

```
【用户画像】
1. 你的目标用户是谁？
   - 是新手还是有经验的？
   - 年龄群体？
   - 专业程度？

2. 用户现在怎么解决这个问题？
   - 用什么现有工具或方法？
   - 手动还是自动？
   - 痛点在哪？

3. 你想做的产品真的帮到他了吗？
   - 现有解决方案哪里不够好？
   - 你的产品能解决什么关键问题？
```

### 商业视角提问

```
【商业模式】
1. 这是一次性需求还是长期需求？
   - 用户会反复使用还是只用一次？
   - 需求的频率是怎样的？

2. 商业模式是什么？
   - 买断制？订阅制？免费+增值？
   - 定价策略？

3. 我们能不能持续做下去？
   - 市场规模有多大？
   - 竞争情况？
```

### 技术视角提问

```
【产品形态】
1. 产品形态是什么？
   - Web/App/小程序/浏览器插件？
   - 平台限制？

2. 能不能快速做出来？
   - MVP 需要多长时间？
   - 技术难点在哪？

3. 开发成本能不能接受？
   - 需要多少人力？
   - 需要外部 API 或服务吗？
```

## 阶段二：落地板补全模板

### 交互细节检查清单

```
【异常流程】
- 点击后报错怎么办？
- 网络失败怎么提示？
- 数据加载中显示什么？
- 加载失败怎么重试？
- 空数据状态怎么展示？

【等待反馈】
- 按钮点击后状态变化？
- 提交中能否重复点击？
- 后台处理时间预估？
- 成功/失败反馈方式？
```

### 状态机模板

```
【状态定义示例：按钮】
- 默认态：可点击，文案"提交"
- 点击态：文案变为"提交中..."，禁用重复点击
- 加载态：显示 loading spinner
- 成功态：短暂显示"提交成功"，然后恢复
- 失败态：显示错误提示，恢复可点击

【状态定义示例：订单】
- 待支付 → 支付中 → 已支付 → 已取消
- 已支付 → 退款中 → 已退款
- 已支付 → 已发货 → 已收货 → 已完成
```

### 字段规范模板

```
【字段规范表示例】

| 字段名 | 类型 | 必填 | 长度 | 校验规则 | 说明 |
|--------|------|------|------|----------|------|
| username | 字符串 | 是 | 2-20字符 | 无特殊字符 | 用户名 |
| email | 字符串 | 是 | 最大100字符 | 邮箱格式校验 | 邮箱 |
| phone | 字符串 | 否 | 11字符 | 手机号格式 | 手机号 |
| amount | 数字 | 是 | - | 大于0 | 金额 |
```

### 文案规范模板

```
【文案规范】
- 按钮文案：动作+对象，如"提交订单"、"保存草稿"
- 提示文案：清晰说明什么操作、什么结果
- 错误文案：说明出了什么问题、怎么解决
- 确认文案：二次确认高风险操作

【文案示例】
- 按钮：["提交", "取消", "确认删除", "保存草稿"]
- 提示：["正在保存...", "提交成功", "正在加载"]
- 错误：["网络错误，请检查网络连接", "表单校验失败，请检查输入"]
- 确认：["确定要删除吗？此操作不可恢复"]
```

### 流程图/线框图说明

```
【核心流程图】
用文字描述关键用户路径：

用户注册流程：
1. 打开APP → 2. 点击注册 → 3. 输入手机号 → 4. 获取验证码
→ 5. 输入验证码 → 6. 设置密码 → 7. 完成注册

【线框图位置】
在 PRD 中标注关键页面：
- 页面1：[首页/概览页] - 位置：section 4.1
- 页面2：[详情页] - 位置：section 4.2
- 页面3：[表单页] - 位置：section 5
```

## 完整 PRD 示例

```markdown
# 自媒体账号定位工具 PRD

## 1. 产品概述

### 1.1 产品背景

帮助自媒体创作者快速找到适合自己的内容定位和账号方向。

### 1.2 核心价值

- 快速生成账号定位建议
- 提供内容方向参考
- 降低创作迷茫感

### 1.3 目标用户

- 刚入门的新手博主
- 遇到瓶颈的成长型创作者

## 2. 用户场景

### 2.1 用户故事

作为[新手博主]，我想[快速找到适合自己的账号定位]，以便[减少探索时间，更快进入正轨]。

### 2.2 使用流程

1. 用户输入基本信息（兴趣方向、擅长领域）
2. 系统分析并生成定位建议
3. 用户选择并细化
4. 输出完整定位报告

## 3. 功能需求

### 3.1 核心功能

- 基础信息收集表单
- AI 分析引擎
- 定位建议生成
- 报告导出

### 3.2 功能流程图
```

输入信息 → AI分析 → 生成建议 → 用户选择 → 输出报告

```

## 4. 交互细节

### 4.1 正常流程
1. 用户打开页面，看到信息收集表单
2. 用户填写兴趣、擅长领域
3. 点击"生成定位"按钮
4. 显示加载状态（2-3秒）
5. 展示定位建议结果

### 4.2 异常情况
- 表单验证失败：红色提示具体错误字段
- 网络错误：显示"网络异常，请重试"
- 分析超时：显示"分析超时，请稍后重试"

### 4.3 状态定义
| 元素 | 状态 | 表现 |
|------|------|------|
| 生成按钮 | 默认 | 可点击，显示"生成定位" |
| 生成按钮 | 点击中 | 禁用，显示"分析中..." |
| 生成按钮 | 成功 | 短暂显示"完成" |
| 生成按钮 | 失败 | 恢复可点击，显示错误toast |

## 5. 字段规范

| 字段名 | 类型 | 必填 | 校验 | 说明 |
|--------|------|------|------|------|
| nickname | 字符串 | 是 | 2-10字符 | 创作者昵称 |
| interests | 多选 | 是 | 至少选1个 | 兴趣领域 |
| strengths | 多选 | 是 | 至少选1个 | 擅长领域 |
| experience | 单选 | 是 | - | 创作经验 |

## 6. 文案规范

- 按钮：["开始定位", "重新生成", "复制报告"]
- 标题：["账号定位建议", "内容方向参考"]
- 提示：["请选择您的兴趣领域", "至少选择一项"]
- 错误：["网络错误，请检查网络后重试"]

## 7. 视觉稿

- 线框图：见附件 /wireframes/account-positioning.png
- 流程图：见附件 /flowcharts/positioning-flow.png

## 8. 技术约束

- 前端：单页应用，可独立运行
- 无后端依赖（AI分析使用模拟数据）
- 支持响应式布局
- 首屏加载 < 3秒
```
