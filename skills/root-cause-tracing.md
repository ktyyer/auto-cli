# Root Cause Tracing -- 根因追踪方法论

> 修 Bug 不是打地鼠。本 Skill 提供结构化的根因追踪方法，从症状出发，
> 逐步深入到根因，确保修复的是问题源头而非表面现象。

---

## 核心原则

1. **先复现再分析** -- 无法复现就无法确认修复
2. **二分法定位** -- 用最小改动范围缩小问题区域
3. **只改一处验证一次** -- 同时改多处会掩盖真正的根因
4. **记录推理链** -- 每一步分析和排除都写下来

---

## 五步根因追踪法

### 步骤 1：症状描述（Symptom）

用结构化格式记录问题：

```markdown
## 症状
- **现象**：[具体的错误信息/异常行为]
- **触发条件**：[什么操作会触发]
- **影响范围**：[影响哪些功能/用户]
- **首次出现**：[什么时候开始出现的]
- **复现步骤**：[1. 2. 3. ...]
```

### 步骤 2：假设生成（Hypothesis）

基于症状，列出所有可能的原因（按可能性排序）：

```markdown
## 假设列表
1. [最可能的原因] -- 理由：...
2. [次可能的原因] -- 理由：...
3. [不太可能但需要排除] -- 理由：...
```

灵感来源：`skills/error-patterns.md` 速查表可以辅助假设生成。

### 步骤 3：二分排除（Bisection）

对每个假设，设计最小验证实验：

```markdown
## 验证实验
| # | 假设 | 验证方法 | 预期结果 | 实际结果 | 结论 |
|---|------|---------|---------|---------|------|
| 1 | ... | [具体命令/操作] | ... | ... | 排除/确认 |
```

**关键技巧**：
- **Git Bisect**：用 `git bisect` 二分查找引入问题的 commit
- **最小复现**：剥离无关代码，创建最小复现用例
- **日志注入**：在可疑位置添加临时日志，观察运行时状态
- **依赖隔离**：逐个锁定依赖版本，排除第三方库变更影响

### 步骤 4：根因确认（Root Cause）

当假设被验证后，用以下格式确认：

```markdown
## 根因确认
- **根因**：[一句话描述根本原因]
- **证据链**：症状 → 排除了 A → 排除了 B → 确认了 C
- **代码位置**：[文件名:行号]
- **为什么之前的版本没问题**：[如适用]
```

### 步骤 5：修复 + 防复发（Fix + Prevention）

```markdown
## 修复方案
- **最小修复**：[只改必要的代码]
- **测试覆盖**：[添加的测试用例]
- **防复发**：[新增的 lint 规则 / CI 检查 / 文档更新]
```

## 常见根因模式

### 模式 1：依赖版本漂移

```
症状：本地正常但 CI 失败
根因：CI 使用了不同版本的依赖
定位：对比 `npm ls` 输出，检查 lock 文件
修复：锁定版本号，使用 `npm ci` 而非 `npm install`
```

### 模式 2：时序竞态

```
症状：偶发性失败，无法稳定复现
根因：异步操作未正确等待或缺少同步机制
定位：检查所有 await 调用、事件监听器、回调顺序
修复：使用 Promise.all 替代多个独立 await；添加错误边界
```

### 模式 3：状态泄漏

```
症状：测试运行顺序影响结果
根因：测试之间共享了可变状态
定位：逐个运行测试（不并行），观察哪些失败
修复：每个测试用 before/after 清理状态；避免全局变量
```

### 模式 4：隐式依赖

```
症状：删除"不相关"代码后功能异常
根因：模块间存在未声明的隐式依赖
定位：搜索被删除代码的引用（Grep 全项目搜索）
修复：显式声明依赖关系；添加 import/require
```

### 模式 5：编码/解码不一致

```
症状：特殊字符显示异常或数据损坏
根因：不同层使用不同的编码格式
定位：检查所有数据流转环节的编码设置
修复：统一使用 UTF-8；在边界层显式转换
```

## 工具辅助

| 工具 | 用途 | 命令 |
|------|------|------|
| git bisect | 二分查找引入问题的 commit | `git bisect start` / `git bisect bad` / `git bisect good` |
| git diff | 对比变更 | `git diff HEAD~1 -- src/file.js` |
| git log | 查看历史 | `git log --oneline -20 -- src/file.js` |
| Grep | 全局搜索引用 | `Grep "targetSymbol" output_mode="content"` |
| Node.js --inspect | 运行时调试 | `node --inspect-brk script.js` |
