# Focus Chain - 任务焦点保持

> 防止 AI 在长对话中"忘记"原始目标

## 触发条件

- 任务预估超过 10 轮对话
- 任务包含多个子步骤
- 用户明确要求"保持专注"或"不要忘记"
- auto-core 评估复杂度为"中等"或"复杂"

---

## 核心机制

### 1. 任务锚点（Task Anchor）

在任务开始时，提取并固定以下信息：

```markdown
## 🎯 任务锚点

- **目标**: [一句话描述最终目标]
- **边界**: [不做什么]
- **完成标准**: [怎样算完成]
- **当前进度**: [x/n 步完成]
```

### 2. 焦点注入（Focus Injection）

每 **6 条消息** 自动注入任务提醒：

```markdown
---
⚡ 焦点提醒：
目标: 重构用户认证模块
进度: 3/7 步 — 正在实现 JWT 验证中间件
待办:
  4. [ ] JWT 验证中间件
  5. [ ] 刷新 token 逻辑
  6. [ ] 旧 session 迁移脚本
  7. [ ] 集成测试
---
```

### 3. 偏离检测（Drift Detection）

当检测到以下信号时，主动纠偏：

- 当前修改的文件不在计划清单中
- 讨论话题偏离原始需求
- 开始"优化"未在计划中的代码
- 引入超出范围的新功能

纠偏提示：

```markdown
⚠️ 检测到可能偏离任务目标
- 当前动作: 修改 database.js 中的连接池配置
- 原始计划: 未包含此文件
- 建议: 回到第 4 步（JWT 验证中间件）
- 如需修改请确认: 是否将此纳入计划？
```

## 焦点文件规范

### 默认路径

- `.aimax/state/focus-chain.md`

### 文件格式

```markdown
# Focus Chain

## 目标
重构用户认证模块，从 session 迁移到 JWT

## 步骤清单
1. [x] 分析现有 session 实现
2. [x] 设计 JWT 方案
3. [x] 实现 JWT 生成
4. [ ] JWT 验证中间件  ← 当前
5. [ ] 刷新 token
6. [ ] 旧 session 迁移
7. [ ] 集成测试

## 约束
- 不修改数据库 schema
- 保持向后兼容 API
- 覆盖率 > 80%

## 消息计数
- 总消息数: 18
- 上次注入: 消息 #18
- 下次注入: 消息 #24
```

---

## 与 auto-core 集成

auto-core 在评估复杂度后自动激活 Focus Chain：

```text
复杂度 >= 中等 → 激活 Focus Chain
复杂度 = 简单 → 不激活（避免额外开销）
```

---

## 与其他命令协作

- `/auto:deep-plan` → 生成计划 → Focus Chain 跟踪执行
- `/auto:loop` → 状态机提供结构化步骤 → Focus Chain 注入提醒
- `/auto:evolve` → 迭代过程中保持目标焦点

---

## 核心原则

1. **非侵入** — 焦点注入不干扰正常编码流程
2. **自适应** — 注入频率根据任务复杂度动态调整
3. **可覆盖** — 用户可以明确要求修改计划，扩展范围
4. **轻量** — 锚点信息尽量精简，不浪费上下文窗口
